NetworkUtils - Документация

Инструмент для проверки скорости интернета и мониторинга доступности сайтов. Построен на FastAPI (Python) и vanilla JS.

Принцип работы

Задержка (Latency)

Браузер отправляет 10 HTTP GET запросов к /api/ping подряд и фиксирует время от запроса до ответа (performance.now()). Верхние 20% выбросов отбрасываются, остальные усредняются.

Скорость загрузки (Download)

Сначала выполняется "разогревающий" запрос на 1 МБ для установки TCP-соединения. Затем с сервера стримится 25 МБ случайных байт. Браузер читает их чанк за чанком через Streams API, подсчитывая скорость в реальном времени.

Используются случайные байты (os.urandom), что позволяет избежать сжатия.

Скорость отдачи (Upload)

Браузер генерирует 10 МБ случайных байт (crypto.getRandomValues) и отправляет их на /api/upload через XMLHttpRequest (чтобы можно было отслеживать прогресс отправки). Скорость вычисляется как байты ÷ время.

Мониторинг доступности

Сервер каждые 30 секунд проверяет список заданных сайтов через httpx и сохраняет последние 240 результатов (≈2 часа) в памяти. Пользователь может добавить собственный URL - его история хранится в localStorage браузера.

API

МетодПутьОписание
GET /api/ping Возвращает {"server_ts": ...}. Используется для измерения задержки.
GET /api/download/{size} Стримит случайные байты. size: 1mb, 10mb, 25mb, 100mb.
POST /api/upload Принимает сырые байты. Возвращает {"received_bytes": N}.
GET /api/uptime/sites Список отслеживаемых сайтов с последним результатом проверки.
GET /api/uptime/history/{id} История проверок для конкретного сайта.
GET /api/uptime/check?url=... Разовая проверка любого URL прямо сейчас.
GET /api/health Проверка состояния сервера. Возвращает {"status": "ok"}.
GET /api/docs Интерактивный Swagger UI (встроен в FastAPI).

Локальный запуск

Нужен только Docker Desktop.

git clone https://github.com/gtbtop/network-utils
cd network-utils
docker compose -f docker-compose.dev.yml up --build

-> http://localhost

Деплой на сервер (HTTPS)

Понадобится сервер с публичным IP-адресом и домен.

1. Привязка домена к серверу

В панели управления доменом нужно найти раздел DNS и создать A-запись: имя @, значение - IP адрес сервера.

2. Копирование проекта на сервер на сервер

rsync -avz ./ user@your-server:~/network-utils/

3. Получение HTTPS-сертификата

Сертификат можно получить бесплатно через Let's Encrypt. На сервере (со своим доменом и почтой):

docker run --rm -p 80:80 \
  -v ~/certbot/conf:/etc/letsencrypt \
  certbot/certbot certonly --standalone \
  --agree-tos --no-eff-email \
  --email you@example.com -d yourdomain.com

4. Запуск

DOMAIN=yourdomain.com docker compose up -d --build

Сайт будет доступен по адресу https://yourdomain.com. Сертификат обновляется автоматически.

Архитектура

Браузер
  │
  ├── GET /            -> nginx -> frontend/index.html
  ├── GET /uptime.html -> nginx -> frontend/uptime.html
  ├── GET /docs        -> nginx -> docs/index.html
  ├── GET /css/* /js/* -> nginx -> frontend/
  │
  └── /api/*           -> nginx -> backend:8000 (FastAPI)
